Paramètres d'application

Vous pouvez passer des informations à votre application Zyllio directement via son URL. Cela permet de piloter l'affichage ou de transmettre des données (comme un nom d'utilisateur ou un ID) dès l'ouverture.

Structure de l'URL

Les paramètres s'ajoutent après l'ID de votre application en commençant par un point d'interrogation (?).

Cas A : Indiquer un écran spécifique (Optionnel)

Si vous souhaitez que l'application s'ouvre sur un écran précis plutôt que sur l'accueil par défaut, placez le nom de l'écran en première position.

  • Exemple : https://zyllio.link/apps/<appId>/?EcranDeVente

Cas B : Passer des données sans changer l'écran

Si vous voulez simplement transmettre des données sans changer l'écran de démarrage, listez vos paramètres directement.

  • Exemple : https://zyllio.link/apps/<appId>/?Nom=Didier

Combiner plusieurs paramètres

Pour cumuler l'écran de démarrage et des données supplémentaires, utilisez le symbole & comme séparateur.

Règle d'or : Si vous spécifiez un écran de démarrage, il doit impérativement être le premier paramètre après le ?.

Exemple complet :

https://zyllio.link/apps/d0f269524cd4/?TableauDeBord&Nom=Didier&Statut=Premium

Dans cet exemple :

  1. L'application force l'ouverture sur l'écran TableauDeBord.
  2. Elle transporte les variables Nom (Didier) et Statut (Premium).

Comment récupérer ces valeurs ?

Pour exploiter ces données dans votre logique No-Code (par exemple pour afficher un message de bienvenue ou filtrer une liste) :

  1. Allez dans la section Application de l'éditeur.
  2. Choisissez la fonction "Paramètre de l'application".
  3. Indiquez le nom de la clé utilisée dans l'URL (ex: Nom).

Gestion des caractères spéciaux (encodage)

Les URLs ne peuvent pas contenir d'espaces ou de caractères accentués directement. Si vos noms d'écrans ou vos valeurs de paramètres en comportent, ils doivent être "encodés".

Les règles de base

  • Les espaces : Remplacez-les par %20.
  • Les accents : Il est fortement recommandé de les éviter dans les noms de vos écrans. Si vous devez passer une valeur avec accent (ex: "Sébastien"), l'URL le transformera souvent automatiquement.
