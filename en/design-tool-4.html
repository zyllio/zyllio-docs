Design tool

At the top of the editor portal, a navigation bar gives access to the main work areas of the application.

Main tabs

The top bar organizes the work into 6 complementary spaces: Screens to build, Transitions to connect, Scenarios to automate, Simulation to test, Data to model, and Settings to configure.

Screens

Main entry point for designing the application interface.

This tab is used to create and organize screens, position visual components, and define the user experience.

Transitions

Used to manage navigation between screens.

It defines links, sequences, and visual transitions that structure the user journey.

Scenarios

Dedicated space for workflows and automation.

It is used to create, edit, and manage scenarios triggered by a user action, a webhook, a schedule, or a data event.

Simulation

Used to test the application in conditions close to real usage.

You can choose a user, change the device type, and restart the simulation to verify the application's behavior.

Data

Area for managing the data model.

This tab allows you to work with the application's database, its tables, columns, and relationships.

Settings

Centralizes the application's general configuration.

It includes application information, plugins, security, and integrations organized by category.

Top-right actions

Publish

Triggers publication of the application so it becomes available in its deployable version.

Open

Displays access to the published application, including the available open and sharing options.
