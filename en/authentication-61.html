Authentication

Protect access to your app with role-based authentication and secure session handling.

Authentication model

  • Define user roles and authorization boundaries.
  • Secure sign-in and sign-out flows across all entry points.
  • Protect sensitive screens with explicit access rules.

Implementation steps

  1. Configure identity source and token lifecycle.
  2. Bind user profile state using Variables.
  3. Add guarded transitions in Screen sequence editor.
  4. Test session expiry and unauthorized access in Simulation.

Security checks

Validate token expiration handling, logout cleanup, and least-privilege access before publishing.
