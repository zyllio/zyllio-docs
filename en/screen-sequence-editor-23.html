Screen sequence editor

Define navigation logic, branching rules, and user progression between screens.

What this editor controls

  • Entry points and default routing.
  • Conditional transitions based on data or user actions.
  • Error and fallback paths for resilient UX.

Implementation pattern

  1. Map the happy path first and ensure every step has one clear next action.
  2. Add conditional branches only after baseline flow is stable.
  3. Validate transitions in Simulation.
  4. Track user completion outcomes from Dashboard.

Navigation checklist

  • No dead-end screens in primary journeys.
  • Back navigation is always predictable.
  • Required data is available before each transition.
