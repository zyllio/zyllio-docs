Screen editor

Build each app screen with reusable components, clear hierarchy, and data bindings.

Build your UI efficiently

  • Use reusable blocks to keep a consistent visual system.
  • Name components based on intent, not position.
  • Bind fields to data sources as early as possible.

Workflow

  1. Create the screen skeleton and key sections.
  2. Add interaction states and visibility conditions.
  3. Connect formulas from Formulas and app state from Variables.
  4. Preview navigation in Screen sequence editor.

Common pitfalls

  • Overloading one screen with too many actions.
  • Using inconsistent labels across similar components.
  • Leaving edge cases untested before simulation.
