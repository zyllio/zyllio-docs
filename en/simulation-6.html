Simulation

The simulator allows you to test mobile applications in a simulated environment, providing the ability to visualize and interact with applications on different mobile phone models. It supports both Android and iOS platforms.

This mobile application simulator is an efficient tool for testing your application's features on various devices and operating systems. It also simplifies the testing process by offering quick authentication through automatic login. Use it to check the performance and user experience of your applications under conditions close to real-life scenarios.

Device Model Selection

The simulator offers a wide range of device models from which you can choose to test your application. You can select popular Android and iOS phones, such as Samsung Galaxy and iPhones.

To change the device model, use the dropdown menu on the left side of the interface. This adjusts the application's display according to the chosen device's specifications, including screen resolution and operating system specifics.

Automatic Authentication

The simulator features an automatic authentication function, allowing you to log into the application without manually entering an email and password.

To use this feature, you need to select a user to authenticate from the dedicated menu. This option simulates quick login to an application without having to enter login credentials each time.

For automatic authentication to work correctly, the authentication table must contain three required columns:

  • Email: The user's email
  • Name: The user's name
  • Password: The user's password

If these columns are not present, the automatic authentication feature cannot be used. Therefore, it is essential to check the table structure before enabling this option.

How to Use the Simulator

Device Selection

Access the dropdown menu at the top left to choose a phone model. Once selected, the simulator will automatically adjust the display to match the appearance of that device.

Automatic Authentication

To enable automatic authentication, select the corresponding option in the "Act as" menu and choose a predefined user. This will simulate the experience of a user logging into the application directly without manually entering their credentials.
