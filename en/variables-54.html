Variables

Manage app state with variables to keep screens synchronized and interactions predictable.

Variable strategy

  • Use local variables for screen-specific temporary values.
  • Use shared variables for cross-screen session context.
  • Reset sensitive variables on logout or flow completion.

Practical setup

  1. Define naming conventions by scope and purpose.
  2. Initialize defaults on entry screens.
  3. Update values through explicit actions and formulas.
  4. Verify state transitions in Screen sequence editor.

State integrity

Document expected variable lifecycle to prevent stale values and hard-to-debug side effects.
