PDF Generation

Generate custom PDF documents like invoices, quotes, contracts, or certificates directly from your Zyllio application using Microsoft Word templates.

Available starting from the Enterprise plan.

Zyllio's Documents integration allows you to dynamically populate Microsoft Word (.docx) templates with variables from your application and compile them into high-quality PDF files instantly, with no coding required.

1. Enable the Documents Integration

Before generating any documents, you need to turn on the integration in your application settings:

  1. Open your application in Zyllio Studio.
  2. Navigate to the Settings tab in the main sidebar.
  3. Click on Integrations.
  4. Find the Documents card and toggle the Enable switch to the active position.

2. Prepare your Word Template (.docx)

You can design your template using Microsoft Word, Google Docs, or LibreOffice. Zyllio uses a simple expression language to map fields:

Simple Fields

To print simple text, numbers, or dates (like a client name or invoice date), insert the field name enclosed in double curly braces:

Example: {{Client Name}} or {{Date}}

Tip: Zyllio automatically normalizes your field names. Spaces, accents, and casing are ignored. For example, {{Nom Client}} and {{nomclient}} will both map to the same field.

Tables & Lists (Loops)

If you need to print a list of items (like invoice rows), you can create a loop. This tells Zyllio to repeat a section or a table row for every item in your list:

  • Start the loop with: {{#items}}
  • Define the fields for each item: {{description}}, {{quantity}}, {{price}}
  • End the loop with: {{/items}}

In a Word Table: Place the opening tag {{#items}} in the first cell of the row, and the closing tag {{/items}} in the last cell of the same row. Zyllio will automatically duplicate the row for each item.

Example in a Word table:

Description Quantity Price
{{#items}}{{description}} {{quantity}} {{price}}{{/items}}

Filtering Items

You can filter list items directly inside the template to display only specific entries:

  • Show only matching items: {{#items[status=Paid]}} only shows items where the status is "Paid".
  • Exclude matching items: {{#items[status!=Paid]}} hides items where the status is "Paid".

Images

You can display a public image stored in an image field by using an image tag in your Word template. The tag name must match your real field or column name, for example {{%image}} or {{%photo}}:

  • Default size: {{%FieldName}} for example {{%image}} or {{%photo}}
  • Custom size: {{%FieldName:50}} for example {{%image:50}} or {{%photo:50}}

The number currently represents a square size in pixels. For example, {{%image:50}} generates a 50x50 image.

Inside a loop: place the image tag in the repeated row or block, for example {{#products}} {{name}} {{%image:50}} {{/products}}.

Important: use a text tag directly in the Word document. Replacing an existing dummy image through alt text is not currently supported.

3. Upload your Template to a Database Table

Since the Zyllio Assets manager is reserved exclusively for images, you must store your Microsoft Word templates (.docx) in a database table:

  1. In your application database, create a table (for example, named Templates).
  2. Add a column of type File to this table.
  3. Upload your .docx file into a row of this table.

4. Configure the "Generate PDF" Action

Now, set up the action that will trigger the document generation (for example, on a button click):

  1. Select your button or trigger element on the screen.
  2. Open the Actions properties.
  3. Add a new step and select Generate PDF (under the Documents category).
  4. Configure the action properties:
    • DOCX template URL: Bind this dynamically to the File column of the database record you uploaded in Step 3 (e.g., Templates.File). Zyllio will automatically resolve it to the absolute file URL.
    • Data fields: Map the variable tags from your Word document to your Zyllio variables or database collections. Open the editor to configure fields using two dedicated buttons:
      • + Field: Adds a single value mapping (text, number, date, etc.). Specify the variable name in your Word template as the key (e.g., clientname) and select a Zyllio screen variable or formula as the value.
      • + List: Adds a list or table mapping. Specify the list/loop name in your Word template as the key (e.g., items) and bind it to a database table or collection.
    • PDF URL: Select or create a text variable (e.g., InvoicePdfUrl) where the URL of the generated PDF will be stored.
    • Loader: Set to True to show a loading spinner on the user's screen while the PDF is being prepared.

Data Fields Options Details

The data property editor offers two distinct types of data rows to map your Word document variables:

  • The + Field button (Simple Values):

    Used to map simple placeholder values like {{name}} or {{date}} in your Word file. You can bind these keys to:

    • Screen input variables (e.g., text fields entered by the user).
    • Dynamic formulas (e.g., tax calculations).
    • System properties (e.g., current logged-in user email).
    • Static hand-written texts.
  • The + List button (Tables and Loops):

    Used to bind structured repeating sections or tables using loop tags like {{#items}}...{{/items}}. You must bind this key to:

    • A database table (e.g., the InvoiceLines table).
    • A filtered collection of records. Zyllio will automatically repeat the section or table row for each record.

5. Display, Download, or Store the Generated PDF

Once Zyllio has generated the PDF and saved its URL into your variable, you can use it in three different ways in your application:

A. Display the PDF

To render the document directly inside your application screen for review:

  1. Add a PDF Viewer component to your screen.
  2. Bind its PDF URL property to the variable where you stored the generated URL (e.g., InvoicePdfUrl).

B. Download or Send the PDF

To export or distribute the document:

  • Attach the PDF URL variable to a download button or an "open URL in browser" action.
  • Send it as an email attachment using Zyllio's email send actions.
  • Forward the URL to external services using Zapier or Make actions.

C. Store the PDF in the Database

To archive the generated file for history or reference:

  1. Add a Create Row or Update Row action in the flow immediately following the PDF generation step.
  2. Select the target database table (e.g., InvoicesHistory).
  3. Bind the table's File or URL column to the variable containing the generated PDF URL (e.g., InvoicePdfUrl). The file will be saved permanently.
